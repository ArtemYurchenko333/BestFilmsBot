# Руководство разработчика - Телеграм-бот рекомендации фильмов

## Архитектура и технические схемы

### Схема потока данных

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           TELEGRAM BOT FLOW                                │
└─────────────────────────────────────────────────────────────────────────────┘

1. ПОЛУЧЕНИЕ СООБЩЕНИЯ
   ┌─────────────┐    HTTP Webhook/Polling    ┌─────────────────┐
   │ Telegram    │ ────────────────────────▶  │ Python Bot      │
   │ User        │                            │ Application     │
   └─────────────┘                            └─────────────────┘

2. ОБРАБОТКА СОСТОЯНИЯ
   ┌─────────────────┐    Handler Routing     ┌─────────────────┐
   │ ConversationHandler │ ──────────────────▶ │ State Function  │
   │ (main.py:240)    │                       │ (start/select_*) │
   └─────────────────┘                        └─────────────────┘

3. СОХРАНЕНИЕ ДАННЫХ
   ┌─────────────────┐    SQL INSERT/UPDATE   ┌─────────────────┐
   │ Handler Function│ ──────────────────────▶ │ PostgreSQL DB   │
   │ save_user_data() │                       │ (users/films)   │
   └─────────────────┘                        └─────────────────┘

4. ГЕНЕРАЦИЯ РЕКОМЕНДАЦИЙ
   ┌─────────────────┐    API Request         ┌─────────────────┐
   │ handle_keywords │ ──────────────────────▶ │ Gemini AI       │
   │ (main.py:208)   │                       │ API             │
   └─────────────────┘                        └─────────────────┘

5. ОТВЕТ ПОЛЬЗОВАТЕЛЮ
   ┌─────────────────┐    Bot Message         ┌─────────────────┐
   │ Response Handler│ ──────────────────────▶ │ Telegram User   │
   │ reply_text()    │                       │ Interface       │
   └─────────────────┘                        └─────────────────┘
```

### Схема состояний ConversationHandler

```
                    /start или "Начать новый поиск"
                                    │
                                    ▼
               ┌─────────────────────────────────────────┐
               │            SELECT_GENRES                │
               │  Отображение кнопок жанров              │
               │  context.user_data['selected_genres']   │
               └─────────────────────────────────────────┘
                                    │
                           Callback: genre_*
                                    ▼
               ┌─────────────────────────────────────────┐
               │            SELECT_YEARS                 │
               │  Отображение кнопок годов               │
               │  context.user_data['selected_years']    │
               └─────────────────────────────────────────┘
                                    │
                           Callback: year_*
                                    ▼
               ┌─────────────────────────────────────────┐
               │           ENTER_KEYWORDS                │
               │  Ожидание текстового сообщения          │
               │  Генерация рекомендаций через Gemini    │
               └─────────────────────────────────────────┘
                                    │
                          Text Message
                                    ▼
               ┌─────────────────────────────────────────┐
               │        ConversationHandler.END          │
               │  Сохранение в БД + опция перезапуска    │
               └─────────────────────────────────────────┘

Навигация:
  ⬅️ back_to_genres: SELECT_YEARS → SELECT_GENRES
  ⬅️ back_to_years:  ENTER_KEYWORDS → SELECT_YEARS
  ❌ cancel: Любое состояние → END
```

### Структура базы данных

```sql
-- Схема отношений таблиц
┌─────────────────────────────────────────────────────────────────┐
│                        DATABASE SCHEMA                         │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────┐                ┌─────────────────────────────┐
│     users       │                │           films             │
├─────────────────┤                ├─────────────────────────────┤
│ id (PK)         │◄──────────────┤│ user_id (FK)               │
│ telegram_username│               ││ id (PK)                    │
│ first_name      │               ││ telegram_username          │
│ last_name       │               ││ user_first_name            │
│ created_at      │               ││ user_last_name             │
└─────────────────┘               ││ user_country               │
                                  ││ genres                     │
                                  ││ years                      │
                                  ││ keywords                   │
                                  ││ film1                      │
                                  ││ film2                      │
                                  ││ film3                      │
                                  ││ gemini_response            │
                                  ││ requested_at               │
                                  ││ user_city                  │
                                  ││ user_phone_number          │
                                  └─────────────────────────────┘

Индексы для оптимизации:
- users.id (PRIMARY KEY)
- films.user_id (FOREIGN KEY + INDEX)
- films.requested_at (INDEX для сортировки по времени)
- films.genres (INDEX для аналитики жанров)
```

## Руководство по развертыванию

### 1. Подготовка окружения

#### Системные требования
```bash
# Минимальные требования
- Python 3.7+
- PostgreSQL 12+
- RAM: 128MB минимум, 512MB рекомендуется
- Диск: 100MB свободного места
- Сеть: Исходящий HTTPS (443)
```

#### Установка зависимостей
```bash
# Клонирование проекта
git clone <repository-url>
cd telegram-movie-bot

# Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

#### Создание .env файла
```bash
# Создать файл .env в корне проекта
cat > .env << EOF
TELEGRAM_BOT_TOKEN3=ваш_токен_от_@BotFather
GEMINI_API_KEY3=ваш_ключ_от_Google_AI_Studio
DATABASE_URL3=postgresql://user:password@host:port/database
EOF
```

#### Получение токенов

**Telegram Bot Token:**
1. Написать @BotFather в Telegram
2. Отправить команду `/newbot`
3. Следовать инструкциям для создания бота
4. Сохранить полученный токен

**Gemini API Key:**
1. Перейти на https://makersuite.google.com/app/apikey
2. Создать новый API ключ
3. Сохранить ключ в переменную окружения

**PostgreSQL Database:**
```sql
-- Создание базы данных
CREATE DATABASE telegram_movie_bot;
CREATE USER bot_user WITH PASSWORD 'secure_password';
GRANT ALL PRIVILEGES ON DATABASE telegram_movie_bot TO bot_user;
```

### 7. Настройка мониторинга

#### Простой healthcheck скрипт
```python
# healthcheck.py
import psycopg2
import os
import sys
from datetime import datetime

def check_database():
    try:
        conn = psycopg2.connect(os.getenv("DATABASE_URL3"))
        cursor = conn.cursor()
        cursor.execute("SELECT 1")
        conn.close()
        return True
    except:
        return False

def check_env_vars():
    required_vars = ["TELEGRAM_BOT_TOKEN3", "GEMINI_API_KEY3", "DATABASE_URL3"]
    return all(os.getenv(var) for var in required_vars)

if __name__ == "__main__":
    print(f"Health check - {datetime.now()}")
    
    if not check_env_vars():
        print("❌ Environment variables missing")
        sys.exit(1)
    
    if not check_database():
        print("❌ Database connection failed")
        sys.exit(1)
        
    print("✅ All checks passed")
```

### 8. Резервное копирование

#### Скрипт backup базы данных
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/home/telegrambot/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="movie_bot_db"
DB_USER="bot_user"

mkdir -p $BACKUP_DIR

# Создание дампа базы данных
pg_dump -U $DB_USER -h localhost $DB_NAME > $BACKUP_DIR/backup_$DATE.sql

# Удаление старых backup'ов (старше 7 дней)
find $BACKUP_DIR -name "backup_*.sql" -mtime +7 -delete

echo "Backup completed: backup_$DATE.sql"
```

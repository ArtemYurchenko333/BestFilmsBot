# Телеграм-бот рекомендации фильмов - Документация API

## Обзор

Это Telegram-бот, который предоставляет персонализированные рекомендации фильмов с использованием ИИ Google Gemini. Бот проводит пользователя через разговорный интерфейс для выбора предпочтений по фильмам (жанр, период времени, ключевые слова) и генерирует топ-3 рекомендации фильмов на основе их критериев.

## Функции базы данных

### `get_db_connection()`

**Описание**: Устанавливает соединение с базой данных PostgreSQL.

**Возвращает**: 
- Объект `psycopg2.connection` при успехе
- `None` при ошибке

**Пример**:
```python
conn = get_db_connection()
if conn:
    # Используем соединение
    cursor = conn.cursor()
    # ... выполняем операции с базой данных
    conn.close()
```

### `create_tables_if_not_exists()`

**Описание**: Создает необходимые таблицы базы данных, если они не существуют.

**Создаваемые таблицы**:
- `users`: Хранит информацию профиля пользователя
- `films`: Хранит запросы и ответы рекомендаций фильмов

**Использование**:
```python
create_tables_if_not_exists()
```

**Схема таблиц**:

#### Таблица Users
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY,
    telegram_username VARCHAR(255),
    first_name VARCHAR(255), 
    last_name VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### Таблица Films  
```sql
CREATE TABLE films (
    id SERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    telegram_username VARCHAR(255),
    user_first_name VARCHAR(255),
    user_last_name VARCHAR(255),
    user_country VARCHAR(255),
    genres TEXT NOT NULL,
    years TEXT NOT NULL,
    keywords TEXT,
    film1 VARCHAR(255),
    film2 VARCHAR(255), 
    film3 VARCHAR(255),
    gemini_response TEXT NOT NULL,
    requested_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    user_city VARCHAR(255),
    user_phone_number VARCHAR(20),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### `save_user_data(user_id, username, first, last)`

**Описание**: Сохраняет или обновляет информацию о пользователе в базе данных.

**Параметры**:
- `user_id` (int): ID пользователя Telegram
- `username` (str): Имя пользователя в Telegram
- `first` (str): Имя пользователя
- `last` (str): Фамилия пользователя

**Возвращает**: None

**Пример**:
```python
await save_user_data(
    user_id=123456789,
    username="ivan_petrov", 
    first="Иван",
    last="Петров"
)
```

### `save_film_request(user_id, genres, years, keywords, gemini_response, **kwargs)`

**Описание**: Сохраняет запрос рекомендации фильма и ответ в базу данных.

**Параметры**:
- `user_id` (int): ID пользователя Telegram
- `genres` (str): Выбранные жанры (через запятую)
- `years` (str): Выбранные временные периоды (через запятую)
- `keywords` (str): Ключевые слова пользователя
- `gemini_response` (str): Сгенерированный ИИ текст ответа
- `film1` (str, опционально): Первый рекомендованный фильм
- `film2` (str, опционально): Второй рекомендованный фильм
- `film3` (str, опционально): Третий рекомендованный фильм
- `username` (str, опционально): Имя пользователя в Telegram
- `first_name` (str, опционально): Имя пользователя
- `last_name` (str, опционально): Фамилия пользователя
- `country` (str, опционально): Страна пользователя
- `city` (str, опционально): Город пользователя
- `phone` (str, опционально): Номер телефона пользователя

**Пример**:
```python
await save_film_request(
    user_id=123456789,
    genres="Боевик, Триллер",
    years="2010-2020",
    keywords="супергерои, взрывы",
    gemini_response="1. Мстители...",
    film1="Мстители",
    film2="Железный человек", 
    film3="Темный рыцарь",
    username="ivan_petrov",
    first_name="Иван",
    last_name="Петров"
)
```

## Основные функции бота

### `extract_film_names(text)`

**Описание**: Извлекает названия фильмов из ответа Gemini ИИ, используя регулярные выражения.

**Параметры**:
- `text` (str): Текст ответа ИИ, содержащий пронумерованные рекомендации фильмов

**Возвращает**: 
- `list`: Список из 3 названий фильмов [фильм1, фильм2, фильм3], с None для отсутствующих записей

**Шаблон**: Соответствует пронумерованным спискам типа "1. Название фильма" или "1. Название фильма: Название фильма"

**Пример**:
```python
response_text = """
1. Мстители: Финал
2. Название фильма: Железный человек
3. Темный рыцарь
"""

films = extract_film_names(response_text)
# Результат: ["Мстители: Финал", "Железный человек", "Темный рыцарь"]
```

## Функции-обработчики

### `start(update, context)`

**Описание**: Точка входа, которая инициирует разговор рекомендации фильмов.

**Параметры**:
- `update` (telegram.Update): Объект обновления Telegram
- `context` (telegram.ext.ContextTypes.DEFAULT_TYPE): Контекст бота

**Возвращает**: Состояние `SELECT_GENRES`

**Функциональность**:
- Сохраняет данные пользователя в базу данных
- Очищает предыдущие данные разговора
- Отображает клавиатуру выбора жанра
- Инициализирует user_data пустыми списками

**Пример использования**:
```python
# Пользователь отправляет: /start
# Ответ бота: "Привет, @username! Выбери жанр:" + кнопки жанров
```

### `select_genres(update, context)`

**Описание**: Обрабатывает выбор жанра и переходит к выбору года.

**Параметры**:
- `update` (telegram.Update): Callback query с выбранным жанром
- `context` (telegram.ext.ContextTypes.DEFAULT_TYPE): Контекст бота

**Возвращает**: Состояние `SELECT_YEARS`

**Функциональность**:
- Сохраняет выбранный жанр в context.user_data
- Отображает клавиатуру выбора временного периода
- Показывает подтверждение выбранного жанра

**Пример работы**:
```python
# Пользователь нажимает: кнопку "Боевик"
# Бот сохраняет: context.user_data['selected_genres'] = ["Боевик"]
# Ответ бота: "Вы выбрали жанр: Боевик. Теперь выбери годы:" + кнопки годов
```

### `select_years(update, context)`

**Описание**: Обрабатывает выбор временного периода и переходит к вводу ключевых слов.

**Параметры**:
- `update` (telegram.Update): Callback query с выбранным временным периодом
- `context` (telegram.ext.ContextTypes.DEFAULT_TYPE): Контекст бота

**Возвращает**: Состояние `ENTER_KEYWORDS`

**Функциональность**:
- Сохраняет выбранный временной период в context.user_data
- Отображает запрос на ввод ключевых слов
- Показывает сводку выбора

**Пример работы**:
```python
# Пользователь нажимает: кнопку "10-е (2010-2020)"  
# Бот сохраняет: context.user_data['selected_years'] = ["10-е (2010-2020)"]
# Ответ бота: "Вы выбрали: Жанр: Боевик, Годы: 10-е (2010-2020). Введите ключевые слова:"
```

### `handle_keywords(update, context)`

**Описание**: Обрабатывает ключевые слова пользователя и генерирует рекомендации фильмов с помощью Gemini ИИ.

**Параметры**:
- `update` (telegram.Update): Текстовое сообщение с ключевыми словами
- `context` (telegram.ext.ContextTypes.DEFAULT_TYPE): Контекст бота

**Возвращает**: `ConversationHandler.END`

**Функциональность**:
- Собирает ключевые слова пользователя
- Формирует запрос для Gemini ИИ
- Генерирует рекомендации фильмов
- Извлекает названия фильмов из ответа
- Сохраняет запрос в базу данных
- Предлагает опцию перезапуска

**Пример работы**:
```python
# Пользователь отправляет: "супергерои, взрывы, экшн"
# Бот формирует запрос: "ТОП-3 фильмов в жанре Боевик, 2010-2020. По ключевым словам: 'супергерои, взрывы, экшн'..."
# Gemini генерирует ответ с 3 рекомендациями фильмов
# Бот отправляет ответ пользователю и сохраняет в базу данных
```

### Обработчики навигации

#### `back_to_genres(update, context)`
**Описание**: Возвращает пользователя к шагу выбора жанра.
**Возвращает**: Состояние `SELECT_GENRES`

#### `back_to_years(update, context)`  
**Описание**: Возвращает пользователя к шагу выбора года.
**Возвращает**: Состояние `SELECT_YEARS`

#### `cancel(update, context)`
**Описание**: Отменяет текущий разговор и очищает данные пользователя.
**Возвращает**: `ConversationHandler.END`

#### `unknown(update, context)`
**Описание**: Обрабатывает неизвестные команды во время разговора.

## Вспомогательные функции

### `main()`

**Описание**: Главная точка входа приложения, которая настраивает и запускает бота.

**Функциональность**:
- Создает таблицы базы данных
- Строит приложение Telegram
- Настраивает обработчик разговора с состояниями и callbacks
- Запускает опрос обновлений

**Конфигурация потока разговора**:
```python
conv_handler = ConversationHandler(
    entry_points=[
        CommandHandler("start", start),
        CallbackQueryHandler(start, pattern="^start_over$")
    ],
    states={
        SELECT_GENRES: [CallbackQueryHandler(select_genres, pattern="^genre_")],
        SELECT_YEARS: [
            CallbackQueryHandler(select_years, pattern="^year_"),
            CallbackQueryHandler(back_to_genres, pattern="^back_to_genres$")
        ],
        ENTER_KEYWORDS: [
            MessageHandler(filters.TEXT & ~filters.COMMAND, handle_keywords),
            CallbackQueryHandler(back_to_years, pattern="^back_to_years$")
        ],
    },
    fallbacks=[CommandHandler("cancel", cancel), MessageHandler(filters.COMMAND, unknown)]
)
```

## Справочник API

### Константы

| Константа | Тип | Описание |
|-----------|-----|----------|
| `SELECT_GENRES` | int | Состояние разговора для выбора жанра (0) |
| `SELECT_YEARS` | int | Состояние разговора для выбора года (1) |
| `ENTER_KEYWORDS` | int | Состояние разговора для ввода ключевых слов (2) |
| `FILM_GENRES` | dict | Сопоставление русских названий жанров с английскими значениями |
| `FILM_YEAR_RANGES` | dict | Сопоставление описаний десятилетий с диапазонами лет |

### Переменные окружения

| Переменная | Обязательная | Описание |
|------------|--------------|----------|
| `TELEGRAM_BOT_TOKEN3` | Да | Токен Telegram Bot API |
| `GEMINI_API_KEY3` | Да | Ключ Google Gemini API |
| `DATABASE_URL3` | Да | Строка подключения PostgreSQL |

### Шаблоны Callback Query

| Шаблон | Обработчик | Описание |
|--------|------------|----------|
| `^genre_` | `select_genres` | Callbacks выбора жанра |
| `^year_` | `select_years` | Callbacks выбора временного периода |
| `^back_to_genres$` | `back_to_genres` | Навигация назад к жанрам |
| `^back_to_years$` | `back_to_years` | Навигация назад к годам |
| `^start_over$` | `start` | Перезапуск разговора |

### Фильтры сообщений

| Фильтр | Обработчик | Описание |
|--------|------------|----------|
| `filters.TEXT & ~filters.COMMAND` | `handle_keywords` | Текстовые сообщения для ключевых слов |
| `filters.COMMAND` | `unknown` | Fallback для неизвестных команд |

### Жизненный цикл запроса

1. **Получение обновления** - Telegram отправляет обновление боту
2. **Маршрутизация** - ConversationHandler определяет соответствующий обработчик
3. **Обработка состояния** - Выполняется функция обработчика для текущего состояния
4. **Взаимодействие с базой данных** - Сохранение/получение данных пользователя
5. **Вызов ИИ** - Отправка запроса к Gemini API (при необходимости)
6. **Ответ пользователю** - Отправка ответа через Telegram API
7. **Переход состояния** - Переход к следующему состоянию разговора

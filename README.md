# Выгрузка данных из 1С в Google Sheets

Этот проект предоставляет инструменты для автоматической выгрузки данных из системы 1С:Предприятие в Google Sheets с поддержкой двух способов подключения к 1С.

## Возможности

- **Два способа подключения к 1С:**
  - OData веб-сервис (кроссплатформенный)
  - COM-объект (только Windows)
- **Автоматическое создание и обновление Google Sheets**
- **Гибкая настройка фильтров и полей для выборки**
- **Поддержка пакетной выгрузки**
- **Подробное логирование процесса**

## Установка

1. Установите Python 3.7 или выше

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Для Windows (если планируете использовать COM-подключение):
```bash
pip install pywin32
```

## Настройка

### 1. Настройка 1С

#### Для OData подключения:
1. В 1С включите публикацию данных через OData
2. Настройте веб-сервер (IIS или Apache)
3. Создайте пользователя с правами доступа к данным

#### Для COM подключения:
1. Убедитесь, что 1С установлена на том же компьютере
2. Зарегистрируйте COM-объект 1С

### 2. Настройка Google Sheets API

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте новый проект или выберите существующий
3. Включите Google Sheets API и Google Drive API
4. Создайте сервисный аккаунт:
   - Перейдите в "IAM & Admin" > "Service Accounts"
   - Нажмите "Create Service Account"
   - Заполните информацию и создайте аккаунт
5. Создайте ключ для сервисного аккаунта:
   - Выберите созданный аккаунт
   - Перейдите в "Keys" > "Add Key" > "Create New Key"
   - Выберите JSON формат
   - Сохраните файл как `google_service_account.json`

### 3. Настройка конфигурации

Отредактируйте файл `config.json`:

```json
{
  "connection_type": "odata",
  
  "onec_odata_url": "http://your-server:port/your-base",
  "onec_username": "your-username",
  "onec_password": "your-password",
  "onec_database": "your-database",
  
  "onec_com_connection_string": "Srvr=\"server\";Ref=\"base\";Usr=\"user\";Pwd=\"pass\";",
  
  "google_credentials_file": "google_service_account.json"
}
```

## Использование

### Базовый пример

```python
from main import OneCToGoogleSheetsExporter

# Создание экспортера
exporter = OneCToGoogleSheetsExporter()

# Экспорт справочника номенклатуры
exporter.export_odata_entity(
    entity_name="Catalog_Номенклатура",
    sheet_name="Номенклатура из 1С",
    select_fields=["Ref_Key", "Description", "Артикул"]
)
```

### Экспорт с фильтрами

```python
# Экспорт продаж за январь 2024
exporter.export_odata_entity(
    entity_name="Document_РеализацияТоваровУслуг",
    sheet_name="Продажи январь 2024",
    filters={"Date": "2024-01-01T00:00:00"},
    select_fields=["Number", "Date", "Контрагент", "СуммаДокумента"]
)
```

### Произвольный запрос через COM

```python
query = '''
ВЫБРАТЬ
    Номенклатура.Наименование КАК Товар,
    СУММА(Продажи.Количество) КАС Количество,
    СУММА(Продажи.Сумма) КАК Сумма
ИЗ
    РегистрНакопления.Продажи КАК Продажи
    ЛЕВОЕ СОЕДИНЕНИЕ Справочник.Номенклатура КАК Номенклатура
    ПО Продажи.Номенклатура = Номенклатура.Ссылка
СГРУППИРОВАТЬ ПО
    Номенклатура.Наименование
'''

exporter.export_com_query(query, "Отчет по продажам")
```

## Основные классы

### OneCConnector
Класс для подключения к 1С с поддержкой OData и COM.

### GoogleSheetsUploader  
Класс для работы с Google Sheets API.

### OneCToGoogleSheetsExporter
Основной класс, объединяющий функциональность экспорта.

## Типичные сущности 1С для OData

- `Catalog_Номенклатура` - справочник номенклатуры
- `Catalog_Контрагенты` - справочник контрагентов  
- `Document_РеализацияТоваровУслуг` - документы продаж
- `Document_ПоступлениеТоваровУслуг` - документы закупок
- `AccumulationRegister_ТоварныеЗапасы_Balance` - остатки товаров

## Устранение проблем

### Ошибка подключения к 1С OData
- Проверьте URL и доступность сервера
- Убедитесь в правильности логина/пароля
- Проверьте настройки публикации OData в 1С

### Ошибка Google Sheets API
- Проверьте правильность файла сервисного аккаунта
- Убедитесь, что API включены в Google Cloud Console
- Предоставьте доступ сервисному аккаунту к таблицам

### COM-подключение не работает
- Убедитесь, что используете Windows
- Установите pywin32: `pip install pywin32`
- Проверьте регистрацию COM-объекта 1С
